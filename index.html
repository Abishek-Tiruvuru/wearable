Hardware

Four sensors.
One tiny board.

Every sensor was chosen for accuracy, I²C compatibility, and low power draw — all running off the ESP32-C3's 3.3V rail.

❤️
SpO₂ & Heart Rate
MAX30102

Red and infrared LEDs measure blood oxygen saturation and detect heartbeats via photoplethysmography. Runs a 100-sample rolling buffer with weighted averaging for stable readings.

I²C 0x57 100Hz sample rate 18-bit ADC 16384 ADC range
🌡️
Body Temperature
DS18B20

1-Wire digital temperature sensor on GPIO 2 with a 4.7kΩ pull-up resistor. Set to 12-bit resolution for 0.0625°C accuracy, reading non-blocking every 2 seconds.

1-Wire GPIO 2 12-bit resolution ±0.5°C accuracy Non-blocking
📐
Fall Detection
MPU6050

6-axis IMU tracking acceleration and gyroscope data. A two-stage algorithm detects impact spikes above 2.4g followed by post-impact stillness to confirm a real fall, eliminating false positives.

I²C 0x68 2.4g threshold Two-stage detection Auto-calibrates
🖥️
OLED Display
SSD1306 128×64

Dual-colour display with a yellow top band (16px) for screen titles and a blue main zone for live data. Five rotating screens with page indicator dots and smooth animations.

I²C 0x3C 128×64 px Yellow + Blue 5 screens

How it works

Rolling buffer + weighted smoothing

The firmware collects 100 IR/Red samples per cycle, shifts out the oldest 25 and fills 25 fresh ones on every loop — giving a new SpO₂ reading every ~1 second instead of every 4.

  • SpO₂ calculated with Maxim's open-source algorithm on the full 100-sample window for maximum accuracy
  • Heart rate uses a 10-beat weighted average where recent beats carry more weight, then smoothed over 5 averages
  • Perfusion Index (AC/DC ratio of IR signal) shown live — above 1% is reliable, above 2% is clinical grade
  • Two-stage fall: impact spike above 2.4g must be followed by stillness within 600ms to confirm
  • DS18B20 reads non-blocking every 2s — conversion runs in background while display updates
Display

Five screens.
Zero clutter.

Each screen uses the yellow zone for the title and blue zone for one large value, one status line, and a progress bar. Page indicator dots show which screen is active.

  redBuf[i-SHIFT] = redBuf[i];

About this project

Built from scratch.
Learned everything.

The Wearable Multi-Parameter Physiological Monitoring System started as a biomedical instrumentation college project — an attempt to pack real clinical-grade sensor readings into a tiny ESP32-C3 wearable that actually works.

The journey involved I²C scanning diagnostics, toolchain reinstalls, iterative signal processing improvements, and hours of refining the OLED display layout pixel by pixel for the dual-colour screen.

The result is a working prototype that reads SpO₂, heart rate, temperature, and detects falls — all updating live on a 128×64 display no bigger than a postage stamp.
